Part B: True / False

  1. Statement: Every link inside <ul> must be placed inside an <li>.

    Answer: True

  2. Statement: A page may have multiple <h1> headings to make text larger.

    Answer: False

  3. Statement: aria-current="page" should be used on the active navigation item.

    Answer: True

  4. Statement: A form is accessible even if it has no <label> elements.

    Answer: False

  5. Statement: KLM predicts execution time for expert, error-free performance.

    Answer: True

Part C: Short Answer

Difference Between Semantic and Generic Elements

A semantic element clearly describes its meaning and role in the webpage structure. For example, the <nav> element explicitly represents a navigation section, which helps both developers and assistive technologies understand its purpose.

In contrast, a generic element such as <div> has no inherent meaning. It is primarily used for grouping content for styling or layout purposes and does not convey any information about the type of content it contains.

Why KLM Does Not Model High-Level Planning and Interpretation

The Keystroke-Level Model (KLM) focuses on predicting task completion time by breaking interactions into low-level physical actions such as keystrokes, mouse movements, and mental preparation.

It does not include high-level planning or interpretation because those cognitive processes vary significantly between users and situations. KLM assumes expert users performing tasks without errors, making it unsuitable for modeling decision-making, learning, or problem-solving behaviors.
