Norman‚Äôs & GOMS Model Comparison

While Norman's model focuses on the psychological stages of interaction (mental state), GOMS provides an engineering approach to predict the time it takes for an expert to perform a task.

Comparative analysis: Norman‚Äôs Stages (Psychological) vs. GOMS (Engineering)
Interaction Element Norman‚Äôs 7 Stages (1988) GOMS Framework (1983) Key Differences
Intent Goal: The high-level state the user wants to achieve. Goal: What the user wants to accomplish (e.g., EDIT-TEXT). Both use "Goal" to define the starting point of any interaction.
Strategy Plan: The mental decision on how to achieve the goal. Methods: Precise sequences of steps/operators to reach a goal. Norman focuses on thinking; GOMS focuses on the procedure.
Decision Making Implicitly handled within the "Plan" stage. Selection Rules: Logic used to choose a method when multiple exist. GOMS explicitly models the "If-Then" logic of expert users.
Physicality Execute: The physical manipulation of the interface. Operators: Low-level motor/mental actions (Press, Point, Mentally Prepare). GOMS breaks execution down into timed units (e.g., 1.1s for Pointing).
Feedback Evaluation: Perceiving and interpreting the system's response. Not Modeled: Assumes the user is an expert who does not make mistakes. Norman's model covers the whole cycle; GOMS stops at execution.
Error Handling Explains errors as Slips or Mistakes. Does not account for errors (Error-free performance). Norman is better for identifying design flaws; GOMS is better for efficiency.

The Components of GOMS

To perform a GOMS analysis, you must identify these four elements:

  • Goals: What the user wants to achieve (e.g., "Delete a paragraph").
  • Operators: The smallest actions performed (e.g., "Click mouse button").
  • Methods: Subroutines of operators (e.g., "Select text using mouse then press backspace").
  • Selection Rules: The choice logic (e.g., "If paragraph is short, use backspace; if long, use select+delete").
